# Graduate

Find ideas in daily notes that have earned their own page. Recurring themes, unresolved wiki-links, substantial insights, and articulated frameworks are signals that something has outgrown its daily note.

## Setup

Read `~/.costack/config` for `NOTES_DIR`. Daily notes should be in a subfolder of NOTES_DIR (e.g., `Daily/` or `journals/`). Detect the convention by listing folders.

## Input

- Optional argument: number of days to scan (default: 14)

## Stage 1: Scan

1. Read the last N days of daily notes from NOTES_DIR.
2. Focus on the **user's own writing** — skip sections clearly generated by AI (automated summaries, context updates, generated reports). Look for original thinking, reflections, decisions, and recurring references.
3. Note any wiki-links, themes, or ideas that appear repeatedly.

## Stage 2: Extract Candidates

Look for four signals that an idea is ready to graduate:

### Signal 1: Unresolved Links
Wiki-links (`[[Some Concept]]`) that point to pages that don't exist yet. These are concepts important enough that the user referenced them by name but haven't created a dedicated page.

- Use the vault's link resolution (obsidian CLI `unresolved` command, or grep for `[[links]]` and check if target files exist) to find these.
- Filter out obvious non-candidates (common words, throwaway references).

### Signal 2: Recurring Themes
Ideas, topics, or concerns mentioned in 3+ daily notes across different days. Repetition signals importance.

- Search for similar phrases, keywords, or concepts across the scanned notes.
- Group related mentions together.

### Signal 3: Substantial Insights
A paragraph or more of original thinking on a single topic within a daily note. Long-form reflection that's currently buried in a daily entry.

- Look for multi-sentence blocks that develop a single idea.
- Skip task lists, logistics, and status updates.

### Signal 4: Decisions or Frameworks
Articulated principles, mental models, or decision frameworks. "My approach to X is..." or "The way I think about Y..."

- These are especially valuable because they represent crystallized thinking that can be referenced later.

### Deduplication

Before proposing any candidate, check whether:
- A page with that name (or a close variant) already exists
- The idea is already captured in an existing page's content
- An existing page could be expanded to include this idea (often better than creating a new page)

## Stage 3: Present Candidates

Display a numbered list. For each candidate:

```
N. **[Idea/concept name]**
   - Signal: [which signal type triggered this]
   - Sources: [[Daily Note 1]], [[Daily Note 2]], ...
   - Key excerpt: "[most representative quote from the notes]"
   - Suggested action: [one of the options below]
```

Suggested actions:
- **Create new page** — the idea is distinct enough to stand alone
- **Add to existing page** — name the page and explain where it fits
- **Dismiss** — explain why (too vague, already covered, not substantial enough)

Sort candidates by signal strength: recurring themes and unresolved links first (strongest signal), then substantial insights, then decisions/frameworks.

## Stage 4: Execute

**NEVER create pages without explicit user approval.** After presenting the candidate list:

1. Wait for the user to indicate which candidates to act on (by number or name).
2. For approved candidates, create or update pages:

   **New pages:**
   - Add appropriate frontmatter (type, tags, date created)
   - Populate with content gathered from the source daily notes
   - Include a "Sources" section at the bottom linking back to the daily notes where the idea originated
   - Add backlinks: update the source daily notes to link to the new page (replace plain text with wiki-link if applicable)

   **Additions to existing pages:**
   - Add a new section or expand an existing one
   - Include source attribution (which daily note, what date)

3. Summarize what was created or modified.

## Rules

- **Approval is mandatory.** Present, then wait. Never create pages speculatively.
- **Quality over quantity.** 2 strong candidates are better than 8 weak ones. If nothing meets the bar, say "nothing ready to graduate" and explain why.
- **Respect the user's voice.** When populating new pages, use the user's original phrasing from daily notes. Don't rewrite their thinking in AI language.
- **Link both directions.** New pages link to source notes, source notes link to new pages. Orphan pages defeat the purpose.
- **Frontmatter consistency.** Follow whatever frontmatter conventions exist in the vault's CLAUDE.md or templates. If none exist, use minimal frontmatter (type, tags, date).

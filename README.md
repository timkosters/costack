# CoStack

**CoStack turns Claude Code into an AI chief of staff.** It scans your communication channels, tracks your commitments, surfaces what you're avoiding, and helps you write like a human.

Most people use Claude Code to write software. CoStack uses it to run your life. It layers a memory system, context pipeline, and thinking skills on top of Claude Code so it can operate as a persistent assistant that actually knows what's going on.

## What It Does

CoStack adds nine skills to Claude Code. Each one puts Claude in a specific cognitive mode.

| Skill | What it does |
|-------|-------------|
| `/scan` | Check configured sources for new information: messages, emails, meetings, tasks. Classifies each signal and updates your knowledge base. |
| `/derive` | Reason about what the new information implies. Detects gaps, infers timelines, identifies priority shifts, spots patterns, tracks follow-through. |
| `/drift` | Compare your stated intentions against your actual behavior over time. Surfaces the projects you keep saying matter but never touch. |
| `/reflect` | Analyze patterns across recent sessions. Propose concrete improvements to your CLAUDE.md instructions. |
| `/deep-reflect` | Search years of old notes for connections to whatever you're working on now. Surfaces forgotten ideas that are suddenly relevant. |
| `/graduate` | Find ideas buried in daily notes that deserve their own page. Promote them with proper metadata and cross-links. |
| `/humanize` | Strip AI-generated patterns from any piece of writing. Runs a 24-pattern checklist covering everything from em-dash overuse to significance inflation. |
| `/bootstrap` | First-run setup interview. Builds your profile, configures sources, establishes your taxonomy of work. |
| `/health` | System diagnostic. Checks file integrity, context freshness, configuration consistency, and knowledge base hygiene. |

The core loop is simple: **scan** your world, **derive** what it means, **act** on what matters.

## Architecture Overview

CoStack is built on four ideas:

**Layered instructions.** Claude Code loads CLAUDE.md files at multiple levels: global (your preferences and rules), project-specific (per-workspace instructions), and auto-loaded memory (hard-won lessons). Each layer serves a different purpose. Global rules are brief and always present. Memory files accumulate specifics. Skills load additional context on demand.

**Structured memory.** Your CLAUDE.md rules sit at the top (loaded every session). Below that, a MEMORY.md index file tracks references to deeper topic files. Below that, your knowledge base (markdown notes) is searchable via grep or semantic search. At the bottom, parsed session transcripts let you search past conversations. Information flows downward as it ages and upward as it proves important.

**A two-phase context pipeline.** Scanning and reasoning are deliberately separate. Phase 1 checks your sources and classifies what's new. Phase 2 reasons about implications. This separation matters because scanning is mechanical (check every channel, miss nothing) while reasoning is creative (connect dots, spot gaps, infer what's unsaid). Mixing them produces worse results at both.

**A work taxonomy.** Everything fits into Workspace > Area > Project > Task. Areas are durable responsibilities that never complete. Projects have end states. Tasks are discrete actions. This hierarchy lets CoStack track commitments at the right level of abstraction.

## Installation

```bash
git clone https://github.com/anthropics/costack.git ~/.claude/skills/costack && cd ~/.claude/skills/costack && ./setup.sh
```

This clones CoStack into Claude Code's skill directory (where skills are automatically discovered) and runs the setup script, which creates `~/.costack/` for state and memory. It does not modify your existing configuration without asking.

## Getting Started

1. **Run `/bootstrap`** in Claude Code. CoStack will interview you about your work, communication tools, and preferences. This builds your initial profile and CLAUDE.md configuration.

2. **Run `/scan`** to pull in current context from your configured sources. CoStack classifies each piece of information and updates your knowledge base.

3. **Run `/derive`** to reason about what the scan found. CoStack identifies gaps, tracks commitments, and flags things that need attention.

From there, CoStack works best as a daily habit. Scan in the morning, derive what matters, work through the day, reflect at the end of the week.

## Configuration

CoStack stores its state at `~/.costack/`. The directory structure:

```
~/.costack/
  config              # Your settings (NOTES_DIR, etc.)
  context-map.md      # Current snapshot of your world (regenerated by /scan)
  memory/             # Topic files, diary entries, reflections
    MEMORY.md         # Index (auto-loaded each session, keep under 200 lines)
    diary/            # Session diary entries
    reflections/      # Reflection outputs
  state/              # Scan timestamps
  templates/          # Page templates for projects, people, etc.
```

**Knowledge base.** CoStack works best with a directory of markdown notes. It supports any structure: Obsidian vaults, Logseq graphs, or plain files. Point it at your notes directory during `/bootstrap`.

**Sources.** CoStack adapts to whatever tools you have connected. If you've configured MCP servers for email, calendar, messaging, or task management, CoStack will scan them. If you haven't, it skips them gracefully. No specific tool is required.

**Customization.** Your CLAUDE.md files are the primary customization surface. CoStack's `/reflect` skill proposes changes based on patterns it observes, but you always approve modifications. The system gets smarter over time without requiring manual tuning.

## Design Philosophy

**Understand first, act second.** Scanning and reasoning are separate phases because they require different modes of thinking. Mixing observation with interpretation produces worse results at both.

**Draft only, never send.** CoStack never sends messages, emails, or calendar invites on your behalf. It never deletes files or data. It drafts, and you decide what goes out. This is a hard constraint, not a default that can be overridden.

**Memory is layered.** Fast-loading rules in CLAUDE.md handle the common cases. Topic files store deeper knowledge loaded on demand. Your knowledge base holds years of context searchable when needed. Session history lets you reference past conversations. Each layer has different loading costs and different shelf lives.

**Skills are thinking modes.** Each skill puts Claude in a specific cognitive role with specific constraints. `/scan` is methodical and completionist. `/derive` is connective and inferential. `/humanize` is editorial and pattern-matching. The same underlying model behaves differently depending on which skill frames the task.

**Self-improving.** The diary/reflect cycle means the system learns from every session. When something goes wrong, `/reflect` traces the problem to its source in the instructions and proposes a fix. Rules get stronger through use, not through manual maintenance.

**Your data stays local.** Everything runs on your machine. CoStack reads from your configured sources and writes to local files. No data is sent to external services beyond what Claude Code itself requires.

## Who This Is For

CoStack is for operators, founders, researchers, and knowledge workers who spend their days across too many communication channels and too many projects. People whose bottleneck is not writing code but keeping track of everything happening around them.

It complements code-focused tools rather than replacing them. If you use Claude Code for software development, CoStack adds a layer on top for everything else: tracking commitments, preparing for meetings, maintaining relationships, and writing clearly.

## Contributing

CoStack is open source and contributions are welcome.

**Good first contributions:**
- New signal classifiers for `/scan` (different source types)
- Additional writing patterns for `/humanize`
- Integrations with knowledge base tools
- Documentation improvements

**Before submitting:**
- Keep skills focused. Each skill should do one cognitive job well.
- Maintain the safety model. Nothing that sends, deletes, or modifies external systems.
- Test with a real workflow, not just synthetic examples.

See `docs/ARCHITECTURE.md` for a detailed system overview.
